007 // Lab Report

VLANs & Router-on-a-Stick

Designed and configured a multi-VLAN enterprise network in Cisco Packet Tracer — progressing through four labs from basic connectivity to full inter-VLAN routing using router-on-a-stick. Simulates real-world departmental network segmentation across IT, HR, and Finance.

Cisco Packet Tracer Cisco IOS CLI Router 2911 Switch 2960 802.1Q Trunking Networking Lab
01 // Objective
Build a multi-department enterprise network with logical separation using VLANs and enable inter-department communication through a single router interface using Router-on-a-Stick (IEEE 802.1Q subinterfaces). The lab progresses across four stages — from basic connectivity to full Layer 3 inter-VLAN routing.

03 // Lab Progression
Four-Stage Build
LAB 1
Basic Network — No VLANs
Foundation
Establish basic connectivity between PCs on a single flat network. All devices on the same subnet — no segmentation.
Router — Interface & IP Setup
enable
configure terminal
interface g0/0
ip address 192.168.1.1 255.255.255.0
no shutdown

✓ ping 192.168.1.20 → SUCCESS — PCs communicate on flat network
LAB 2
VLAN Segmentation — Switch Only
Layer 2
Separate PCs into isolated VLANs at the switch level. No inter-VLAN routing yet — communication between VLANs is expected to fail.
Switch — Create VLANs & Assign Ports
vlan 10
vlan 20
vlan 30
exit

interface fa0/1
switchport mode access
switchport access vlan 10

interface fa0/2
switchport mode access
switchport access vlan 20

interface fa0/3
switchport mode access
switchport access vlan 30
✗ ping 192.168.20.10 → FAIL (expected) — VLANs isolated, no routing
LAB 3
VLAN Trunking — Switch to Router Link
Layer 2
Configure the uplink port between the switch and router as a trunk to allow all VLAN traffic to travel across a single physical link using 802.1Q tagging.
Switch — Configure Trunk Port
interface fa0/24
switchport mode trunk
! Allows tagged frames for all VLANs to pass to the router
LAB 4
Router-on-a-Stick — Inter-VLAN Routing
Final Lab
Enable communication between all three VLANs through a single router physical interface using subinterfaces — one per VLAN — each with 802.1Q encapsulation.
Router — Enable G0/0 & Configure Subinterfaces
interface g0/0
no shutdown
exit
! VLAN 10 — IT Department
interface g0/0.10
encapsulation dot1Q 10
ip address 192.168.10.1 255.255.255.0
! VLAN 20 — HR Department
interface g0/0.20
encapsulation dot1Q 20
ip address 192.168.20.1 255.255.255.0
! VLAN 30 — Finance Department
interface g0/0.30
encapsulation dot1Q 30
ip address 192.168.30.1 255.255.255.0
PC Gateway Updates
PC1 (VLAN 10)  → Gateway: 192.168.10.1
PC2 (VLAN 20)  → Gateway: 192.168.20.1
PC3 (VLAN 30)  → Gateway: 192.168.30.1
✓ ping 192.168.20.10 → SUCCESS — IT to HR
✓ ping 192.168.30.10 → SUCCESS — IT to Finance
