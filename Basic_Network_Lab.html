008 // Lab Report

Basic LAN & Routed Network

Foundational networking lab progressing from a simple switch-based LAN to a fully routed two-network topology. Covers MAC vs IP addressing, subnet boundaries, and how routers enable communication between separate networks.

Cisco Packet Tracer Switch 2960 Router 2911 Networking Lab
01 // Objective
Understand how devices communicate on a local network, why subnet boundaries matter, and how a router enables traffic between two separate networks. Progress from a flat LAN to a routed topology.
02 // Part 1 — Basic Switch LAN
Two PCs on One Switch
  PC0 (192.168.10.10) ──── Switch ──── PC1 (192.168.10.20)
  
  Same subnet: 192.168.10.0/24
  No gateway needed — switch handles local forwarding
PC IP Configuration
PC0:
IP: 192.168.10.10  Mask: 255.255.255.0
PC1:
IP: 192.168.10.20  Mask: 255.255.255.0
✓ ping 192.168.10.20 → SUCCESS
Optional — Verify switch MAC table
show mac address-table
Shows how the switch learns and stores MAC addresses automatically
03 // Failure Scenario
Different Subnets — No Router
Changed PC1 to 192.168.20.x — ping failed. Devices were on different networks with no router present. A switch cannot route between subnets — it only forwards within the same network using MAC addresses.
04 // Part 2 — Adding a Router
Two Networks Connected via Router
  PC0 (192.168.10.10)            PC1 (192.168.20.10)
       |                               |
    Switch0                         Switch1
       |                               |
  Router G0/0 (192.168.10.1) ← → Router G0/1 (192.168.20.1)
Router — Configure Both Interfaces
enable
configure terminal

interface g0/0
ip address 192.168.10.1 255.255.255.0
no shutdown
exit

interface g0/1
ip address 192.168.20.1 255.255.255.0
no shutdown
exit
