009 // Lab Report

VLAN Segmentation — Layer 2 Isolation

Configured VLANs on a Cisco 2960 switch to logically segment four PCs into two separate departments — Sales and HR — demonstrating how VLANs create isolated broadcast domains on a single physical switch.

Cisco Packet Tracer Cisco IOS CLI Switch 2960 Networking Lab
01 // Objective
Create two VLANs on a single switch to separate Sales and HR departments. Demonstrate that devices within the same VLAN can communicate, while devices in different VLANs cannot — without a router or Layer 3 switch.
02 // Topology
         [ Switch 2960 ]
    Fa0/1  Fa0/2  Fa0/3  Fa0/4
      |      |      |      |
    PC0    PC1    PC2    PC3

  (SALES) (SALES)  (HR)   (HR)

03 // Configuration
Switch — Create named VLANs
enable
configure terminal

vlan 10
name SALES
exit

vlan 20
name HR
exit
Switch — Assign ports to VLANs
! VLAN 10 — Sales: Fa0/1, Fa0/2
interface range fa0/1-2
switchport mode access
switchport access vlan 10
exit

! VLAN 20 — HR: Fa0/3, Fa0/4
interface range fa0/3-4
switchport mode access
switchport access vlan 20
exit
Verify — show vlan brief
show vlan brief

VLAN  Name    Ports
10    SALES   Fa0/1, Fa0/2
20    HR      Fa0/3, Fa0/4
04 // Testing
Connectivity tests from PC0
Same VLAN (Sales → Sales):
ping 192.168.10.20
✓ SUCCESS — same VLAN, switch forwards normally

Cross VLAN (Sales → HR):
ping 192.168.20.10
✗ FAIL — different VLAN, no router present
