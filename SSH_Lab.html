012 // Lab Report

SSH & Secure Remote Access

Configured SSH version 2 on a Cisco router for encrypted remote management — RSA key generation, local user authentication, VTY line restriction to SSH-only, and a security banner. Replaces insecure Telnet with production-grade encrypted access.

Cisco Packet Tracer Cisco IOS CLI SSH v2 RSA 2048-bit Security Lab
01 // Objective
Configure SSH version 2 on a Cisco router to replace Telnet with encrypted remote access. Set up RSA key pair, local user credentials, restrict VTY lines to SSH only, and add a security banner — matching production network security standards.
02 // SSH vs Telnet
FeatureTelnetSSH v2
EncryptionNone — plaintextAES encrypted
AuthenticationPassword onlyUsername + password
Port2322
Key ExchangeNoneRSA 2048-bit
Use CaseLab / legacy onlyProduction standard
03 // Configuration
Step 1 — Hostname & Domain Name (required for RSA key)
enable
configure terminal
hostname R1
ip domain-name mylab.local
Step 2 — Generate RSA Key Pair (2048-bit)
crypto key generate rsa modulus 2048
! Generates the RSA keys needed for SSH encryption
! 2048-bit is the minimum recommended for production
Step 3 — Create Local User & Enable SSH v2
username admin privilege 15 secret admin123
ip ssh version 2
! privilege 15 = full access
! SSH v2 is more secure than v1
Step 4 — Secure Console & VTY Lines
! Console line — require password for direct access
line console 0
password cisco123
login
exit

! VTY lines — SSH only, use local user database
line vty 0 4
transport input ssh
login local
exit

! Enable secret — encrypted privileged mode password
enable secret admin123
Step 5 — Security Banner & Save
banner motd #
Unauthorized Access Prohibited
#

end
copy running-config startup-config
04 // Connecting via SSH
From PC — SSH connection flow
ssh -l admin 192.168.2.1
! Prompted for password: admin123
! Banner displays: "Unauthorized Access Prohibited"
! Lands in privileged EXEC mode (privilege 15)
R1#
05 // Verification
Verify SSH and configuration
show running-config     Full config including SSH settings

show ip interface brief  Interface status
show users               Active sessions
show vlan brief          VLAN status
show ip route            Routing table
