006 // Lab Report

DHCP, DNS, Web Server & Telnet

A complete small office network built in Cisco Packet Tracer — router-based DHCP for automatic IP assignment, a dedicated DNS server resolving the mylab.com domain, an HTTP web server, and Telnet remote management configured on the router with VTY lines and two-level authentication.

Cisco Packet Tracer Cisco IOS CLI Router 2911 Switch 2960 Server-PT Networking Lab
01 // Objective
Configure a Cisco 2911 router as a DHCP server to automatically assign IP addresses to three client PCs on the 192.168.10.0/24 network. The router acts as both the default gateway and DHCP server. A static server at 192.168.10.2 is excluded from the pool and configured as the DNS server for all clients.

04 // Router Configuration
Cisco IOS Commands
Step 1 — Enter configuration mode & set hostname
Router> enable
Router# configure terminal
Router(config)# hostname DHCP_Router
Step 2 — Configure router interface G0/0
DHCP_Router(config)# interface gigabitEthernet 0/0
DHCP_Router(config-if)# ip address 192.168.10.1 255.255.255.0
DHCP_Router(config-if)# no shutdown
DHCP_Router(config-if)# exit
Step 3 — Exclude static addresses from the pool
DHCP_Router(config)# ip dhcp excluded-address 192.168.10.1 192.168.10.2
! Excludes router (.1) and DNS server (.2) from DHCP assignment
Step 4 — Create the DHCP pool
DHCP_Router(config)# ip dhcp pool AD_LAB_POOL
DHCP_Router(dhcp-config)# network 192.168.10.0 255.255.255.0
DHCP_Router(dhcp-config)# default-router 192.168.10.1
DHCP_Router(dhcp-config)# dns-server 192.168.10.2
DHCP_Router(dhcp-config)# exit
Step 5 — Save configuration
DHCP_Router(config)# end
DHCP_Router# copy running-config startup-config
! Persists config across reloads

08 // DNS & Web Server
Server Configuration
The dedicated Server-PT at 192.168.10.2 runs both DNS and HTTP services — handling name resolution for the lab domain and hosting the internal web page.
Step 1 — Set static IP on Server-PT
Server → Desktop → IP Configuration → Static
IP Address:      192.168.10.2
Subnet Mask:     255.255.255.0
Default Gateway: 192.168.10.1
DNS Server:      192.168.10.2
Step 2 — Enable DNS Service & add A records
Server → Services → DNS → ON

Add A Records:
Name: www.mylab.com       Type: A   Address: 192.168.10.2
Name: server.mylab.com    Type: A   Address: 192.168.10.2
Step 3 — Enable HTTP Web Server
Server → Services → HTTP → ON
(Optional: edit index.html under Edit File to customise the hosted page)

10 // Telnet Remote Access
Remote Router Management via Telnet
Telnet allows you to remotely access and configure the router from any PC on the network — no physical console cable needed. VTY (Virtual Terminal) lines control how many simultaneous remote sessions are allowed.
Router — Configure VTY lines & passwords
configure terminal
line vty 0 4
! Enter VTY config — allows up to 5 simultaneous connections
password cisco123
! Telnet login password
login
exit
enable secret admin123
! Password required to enter privileged EXEC (#) mode
end
copy running-config startup-config
PC — Connect to router via Telnet
telnet 192.168.10.1
Password: cisco123
! Now in User EXEC mode
Router> enable
Password: admin123
! Now in Privileged EXEC mode — full access
Router#
