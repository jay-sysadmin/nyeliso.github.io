011 // Lab Report

LAN-to-Internet Connectivity

Built a LAN-to-server network simulating internet-facing communication — PC on a private 192.168.1.0/24 network routing through a router to reach a public server at 200.0.0.10. Includes real troubleshooting: incorrect gateway diagnosis and fix.

Cisco Packet Tracer Cisco IOS CLI Router 1941 Switch 2960 Networking Lab
01 // Objective
Connect a LAN PC (192.168.1.10) to an external server (200.0.0.10) through a router acting as the default gateway. Configure correct IP addressing on both sides, verify router interface status, and troubleshoot gateway misconfiguration.

04 // Troubleshooting
Common Mistake — Wrong Gateway
Error: PC was configured with the wrong default gateway. Without the correct gateway (192.168.1.1), the PC cannot route packets outside its LAN — ping to 200.0.0.10 fails with "Destination Host Unreachable."

Fix: Set PC gateway to 192.168.1.1 (the router's LAN interface). This is the critical step — the gateway must always be the router interface on the same subnet as the PC.
Troubleshooting order
1. Check PC IP + Gateway (most common failure point)
2. Check router interface status (show ip interface brief)
3. Check server IP + Gateway
4. Check cable connections
5. Test step-by-step — never all at once
